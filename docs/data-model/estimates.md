# Estimate

## Purpose

The Estimate entity represents a preliminary repair estimate generated for a customer's requested service.

Its purpose is to provide customers with a realistic starting point for pricing and expected repair duration before a vehicle is inspected by a technician.

An estimate is informational only and should never be considered a final quote.

---

# Description

The AI uses estimate information to:

- Provide preliminary pricing.
- Estimate repair duration.
- Help customers make informed decisions.
- Determine appointment length.
- Transition customers into the booking process.

---

# Core Fields

| Field | Required | Description |
|---------|:--------:|-------------|
| Estimate ID | Optional | Internal unique estimate identifier. |
| Customer ID | Optional | Associated customer. |
| Vehicle ID | Optional | Associated vehicle. |
| Service ID | Yes | Requested service. |
| Estimated Starting Price | Optional | Preliminary starting price. |
| Estimated Duration | Optional | Expected repair time. |
| Currency | Optional | Currency used for pricing. |
| Estimate Date | Optional | Date the estimate was generated. |
| Disclaimer | Yes | Explains that pricing is preliminary. |
| Notes | Optional | Additional estimate information. |

---

# Estimate Characteristics

Every estimate should be treated as:

- Preliminary
- Informational
- Non-binding
- Subject to inspection
- Subject to technician diagnosis

The AI should clearly communicate these limitations whenever discussing pricing.

# Conversation Usage

Typical conversation flow:

Customer requests pricing

↓

Collect vehicle information

↓

Generate estimate

↓

Present estimate

↓

Explain pricing limitations

↓

Offer appointment

The AI should never present an estimate as a guaranteed repair price.

---

# Validation Rules

The AI should:

- Wait for the Service Estimates workflow to complete.
- Present only approved estimate values.
- Never invent pricing.
- Never modify estimate values manually.

If no estimate is available, explain that pricing requires a vehicle inspection.

---

# Pricing Disclaimer

Whenever an estimate is presented, the AI should communicate that:

- The estimate is a starting point.
- Final pricing depends on inspection.
- Additional repairs may affect the final cost.

This explanation should be brief, clear, and consistent.

# Engineering Notes

The Estimate entity exists to support customer decision-making and appointment scheduling.

It should never be treated as a finalized repair quote or invoice.

Future implementations may enrich estimates with labor rates, parts pricing, confidence scores, or regional pricing adjustments, while preserving the principle that all estimates remain preliminary until confirmed by a technician.
